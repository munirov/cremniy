## 1. Описание модулей

### 1.1. Определение

Модуль - это встроенный расширяемый компонент Cremniy, решающий конкретную задачу и работающий внутри программы Cremniy.  

Модули являются частью исходного кода и поставляются вместе с Cremniy. Они не подгружаются динамически из внешних источников и **не поддерживают сторонние пользовательские расширения** - это сделано для:

1. обеспечения отсутствия рисков безопасности, связанных с внешними плагинами
2. централизованного контроля качества модулей
3. поддержания строго сфокусированного набора модулей, ориентированного исключительно на задачи low-level разработки

Добавление новых модулей возможно **только через внесение изменений в репозиторий Cremniy**. Для этого необходимо **создать Pull Request** и **пройти процесс ревью** с последующим одобрением участников проекта.

Cremniy предоставляет **тщательно отобранные** низкоуровневые инструменты со **строгими гарантиями качества** (Никакого хаоса с плагинами, только проверенные низкоуровневые модули).

### 1.2. Виды

В Cremniy существует 3 вида модулей:  
- Tab (Вкладка)
- Window (Окно)
- Reference (Справочник)

Каждый вид модуля имеет собственное место в интерфейсе и отвечает за определённую область функциональности Cremniy. Об этом далее.

#### 1.2.1. Tab (Вкладка)
Данный вид модуля отображается в основном рабочем пространстве как вкладка для работы с файлом. Такие модули позволяют в разных представлениях посмотреть на файл  
Как уже было сказано, этот вид модуля может работать с открытым файлом, изменять его, читать, анализировать.  
Данный вид модулей располагается в `Menu Bar -> Tools -> Tabs`.

#### 1.2.2. Window (Окно)
Данный вид модуля отображается как отдельное окно и выполняет работу не связанную с текущим открытым файлом.   
То есть этот вид модуля не взаимодействует с открытым файлом и не выполняет операций его редактирования, чтения или анализа.  
Данный вид модулей располагается в `Menu Bar -> Tools -> Windows`.

#### 1.2.3. Reference (Справочник)
Данный вид модуля отображается как отдельное окно и занимается тем, что показывается пользователю какую-либо полезную справочную информацию. Этот вид модуля не занимается какой-либо работой и вычислениями.  
Данный вид модулей располагается в `Menu Bar -> References`.

### 1.3. Расположение

Все модули в Cremniy хранятся внутри исходного кода проекта и структурируются по их виду.  
Базовая директория модулей:  `src/Modules/`  
Внутри неё модули разделяются на виды:  
- `src/Modules/Tabs/`
- `src/Modules/Windows/`
- `src/Modules/References/`

### 1.4. Структура

Каждый модуль реализуется как отдельная сущность внутри своей категории и содержит всю необходимую логику для выполнения своей задачи.  
Пример структуры:  
- `src/Modules/Tabs/ExampleTabModule`
- `src/Modules/Windows/ExampleWindowModule`
- `src/Modules/References/ExampleReferenceModule`

## 2. Добавление модулей

### 2.1. Кратко

Если кратко, то для добавления нового модуля необходимо:

- Создать директорию модуля в соответствующей её виду категории (`Tab`, `Window`, `Reference`)
- Создать основной класс модуля, наследующий определенный класс в зависимости от категории
- Реализовать логику работы модуля
- В корне директории модуля создать файл `CMakeLists.txt` и там описать все исходные файлы модуля
- Регистрация модуля в `ModuleManager`. Зависит от категории

Подробности далее.

### 2.2. Tab

Модули этого типа наследуются от класса `TabBase`
> Расположение класса: `src/core/modules/TabBase.h`

Каждый модуль должен регистироваться в `ModuleManager` (`src/core/modules/ModuleManager.h`) в главном файле модуля с помощью функции `registerTab`:

``` cpp
static bool registered = []() {
    ModuleManager::instance().registerTab("ExampleName", "ExampleGroup", []() { return new ExampleTab(); }, 200);
    return true;
}();
```

Аргументы (по порядку):
- `name` - название инструмента.
- `group` - группа к которой относится инструмент. Если группы нет - можно оставить пустую строку "".
- `creator` - лямбда возвращающая модуль.
- `position` - позиция инструмента если он в группе `always`, иначе можно оставить любое значение. 

### 2.3. Window

Модули этого типа наследуются от класса `WindowBase`
> Расположение класса: `src/core/modules/WindowBase.h`

Каждый модуль должен регистироваться в `ModuleManager` (`src/core/modules/ModuleManager.h`) в главном файле модуля с помощью функции `registerWindow`:

``` cpp
static bool registered = []() {
    ModuleManager::instance().registerWindow("ExampleName", "ExampleGroup", []() { return new ExampleWindow(); });
    return true;
}();
```

### 2.4. Reference

Модули этого типа наследуются от класса `ReferenceBase`
> Расположение класса: `src/core/modules/ReferenceBase.h`

Каждый модуль должен регистироваться в `ModuleManager` (`src/core/modules/ModuleManager.h`) в главном файле модуля с помощью функции `registerReference`:

``` cpp
static bool registered = []() {
    ModuleManager::instance().registerReference("ExampleName", "ExampleGroup", []() { return new ExampleReference(); });
    return true;
}();
```
