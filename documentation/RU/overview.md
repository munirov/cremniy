# Cremniy — обзор

## Что это

Cremniy — среда разработки для низкоуровневой работы: бинарники, память, системный код.
Десктоп-приложение на Tauri + React, один процесс, без переключения между инструментами.

## Видение

Один глубокий низкоуровневый верстак, где всё связано: байт ↔ инструкция ↔ строка исходника ↔
адрес в памяти ↔ живой регистр. Одна модель, тычок в любом представлении уводит во все остальные.
Не только статика по мёртвому файлу — а живой запущенный процесс в реальном времени.

Веб-стек выбран в том числе для того, чтобы ИИ мог работать в Cremniy так же, как человек —
не через кнопки, а через команды, которые эти же кнопки вызывают. См.
[AGENT_CONTROL](../architecture/AGENT_CONTROL.md).

## Принципы

- **Всё в одном.** Всё необходимое для низкоуровневой работы — в одной среде.
- **Минимализм.** Ничего лишнего — и в UI, и в наборе фич.
- **Одна модель, много представлений.** Байт, инструкция, строка исходника, адрес, регистр —
  разные виды одного и того же. Навигация в обе стороны.
- **Статика и рантайм.** Смотрим файл на диске; смотрим живой процесс; тот же UI, те же команды.
- **Способность сначала команда, потом кнопка.** Каждая UI-способность — именованная команда в
  `window.cremniy`. Кнопка её зовёт. Человек ходит кнопками, скрипты и ИИ ходят командами — одна
  логика, одни данные.
