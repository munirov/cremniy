# BMFP — слои фронтенда (Cremniy)

В проекте используется **BMFP** только внутри пакета [`source/frontend/`](../../source/frontend/README.md).

## Слои (`source/frontend/src/`)

| Слой | Каталог | Назначение |
|------|---------|------------|
| **boundary** | `boundary/` | Страницы, вёрстка, презентационные компоненты; **без** прямых вызовов ОС — через инфраструктуру. |
| **domain** | `domain/` | Типы, DTO, чистая логика смысла для UI (без импортов Tauri). |
| **shared** | `shared/` | Токены, общие утилиты без доменного смысла. |
| **infrastructure** | `infrastructure/` | Обёртки `invoke`, доступ к хранилищу — **тонкий** транспорт. |

**Направление зависимостей:** boundary → domain, shared; domain → shared; infrastructure → shared и контракты domain.

## Уровень выше

Структура **всего репозитория** (десктоп, доки, CI) — в документе **[BMAP](../architecture/BMAP.md)**.

Отдельный HTTP-**бэкенд** в репозитории (если появится) описывал бы **BMSP**; десктоп остаётся BMFP + нативная оболочка по BMAP.
